# Contributing

Thanks for wanting to add to the SOC Analyst Toolkit.

## The one rule

Every tool in this list must be something a SOC analyst or detection engineer would **actually reach for on shift**. No filler, no link farms, no dead links.

## How to add a tool

1. Fork the repo and create a branch.
2. Add your tool to the correct section table in `README.md`.
3. Use this format:

   ```
   | [Tool Name](https://link) | One-line description of what it does and why an analyst uses it. | — |
   ```

4. Keep the description to one line. Say what it does, not marketing copy.
5. If the tool ties to a documented lab, link the writeup in the **Lab** column. Otherwise use `—`.

## Quality bar

- The tool must be free, open-source, or have a usable free tier.
- The link must go to the official project page, not a blog or aggregator.
- No duplicates. Check the section before adding.

## Submitting

Open a pull request with a short note on why the tool earns its place. PRs that pass the one rule get merged fast.
